\section{Executive Summary}
% ============================================================

\begin{infobox}[title={One-Line Pitch}]
AlphaStream India is a Bloomberg-style AI investment terminal that turns ET Markets data
into actionable, real-time signals for India's 14~crore+ retail investors.
\end{infobox}

\medskip

AlphaStream India is a production-grade investment intelligence platform built on
\textbf{Pathway streaming RAG} and a \textbf{13-agent AI reasoning pipeline}.
It addresses a fundamental gap in the Indian retail investment ecosystem: institutional-grade
analytics are locked behind expensive Bloomberg or Reuters terminals, leaving the vast
majority of retail investors dependent on WhatsApp tips and delayed news.

\subsection*{Key Differentiators}

\begin{itemize}[leftmargin=1.6em, itemsep=3pt]
  \item \textbf{Pathway Adaptive RAG} --- news-to-recommendation latency under 2 seconds via
        incremental streaming (not batch processing).
  \item \textbf{13 specialised AI agents} --- each focuses on one analytical dimension
        (Sentiment, Technical, Risk, FII/DII Flow, Filing, Insider, Anomaly, \ldots);
        the Decision Agent fuses them all into a single Alpha Score.
  \item \textbf{Text2SQL NLQ} --- grounded answers from a live DuckDB database, not
        LLM hallucination. Ask in plain English; get SQL-verified facts.
  \item \textbf{India-first context} --- \Rs{}~currency, IST timezone, NSE/BSE bare tickers,
        FII/DII flows, BSE corporate filings, Groww fundamentals.
  \item \textbf{Bloomberg-style 5-tab terminal} --- Overview, Signals, Global Intel,
        Company, Portfolio --- each tab maps to a distinct investor workflow.
\end{itemize}

\section{Problem Statement}
% ============================================================

\subsection{The Gap}

India has \textbf{14 crore+ demat accounts} (CDSL + NSDL, 2024) yet the tools available
to retail investors are dramatically inferior to those used by institutional traders:

\begin{itemize}[leftmargin=1.6em, itemsep=3pt]
  \item \textbf{Stale data} --- most free platforms update prices every 5--15 minutes;
        institutional desks consume tick data in real time.
  \item \textbf{No signal aggregation} --- a retail investor checks RSI manually on one screen,
        reads FII data on NSDL's PDF portal, and hunts for insider trades on NSE's filing
        search --- three separate workflows with no synthesis.
  \item \textbf{LLM tools are ungrounded} --- existing "AI chatbots" for investing answer
        questions with trained weights, not live market data. They hallucinate stock prices
        and miss earnings announced an hour ago.
  \item \textbf{2+ hours of daily research} --- surveys consistently show active retail
        investors spend 2--3 hours per day on research that could be automated.
\end{itemize}

\subsection{Our Solution: Live AI}

AlphaStream implements the \emph{Live AI} paradigm:

\subsection{Data Flow}

\begin{enumerate}[leftmargin=1.8em, itemsep=4pt]
  \item \textbf{Ingest}: The Pathway \texttt{ConnectorSubject} polls 5 news APIs in
        parallel every 60 seconds; articles are written to \texttt{data/articles/}.
  \item \textbf{Index}: \texttt{pw.io.fs.read} detects new files and the
        \texttt{DocumentStore} chunks, embeds (all-MiniLM-L6-v2), and indexes them
        into UsearchKnnFactory --- all incrementally, with no full rebuild.
  \item \textbf{Callback}: \texttt{pw.io.subscribe} fires \texttt{on\_new\_article()} on each
        new row, triggering DuckDB persistence and WebSocket broadcasts.
  \item \textbf{Recommend}: On a \texttt{POST /recommend} request, 12 specialist agents
        run in parallel (via \texttt{asyncio.gather}), and the Decision Agent synthesises
        their scores into a single BUY/HOLD/SELL recommendation with an Alpha Score.
  \item \textbf{Deliver}: The response is pushed to the React dashboard over WebSocket
        (\texttt{ws://localhost:8000/ws/stream/\{ticker\}}) in under 50 ms.
\end{enumerate}

\subsection{Fallback Strategy}

When \texttt{ENABLE\_PATHWAY=false} (default for quick setup), the system uses a
synchronous \texttt{RAGPipeline} backed by ChromaDB with in-memory embeddings.
All API endpoints behave identically --- only the latency profile changes.

The NLQ (Natural Language Query) system converts free-form investor questions into
SQL queries against the live DuckDB analytics database, returning grounded answers.

\begin{center}
\small
\begin{tabular}{C{0.5cm} L{2.8cm} L{9.5cm}}
  \toprule
  \textbf{\#} & \textbf{Node} & \textbf{Responsibility} \\
  \midrule
  1 & Guardrail     & Rejects off-topic / harmful queries; returns canned response \\
  2 & Enrich        & Optional web search for context not in DB (e.g., recent events) \\
  3 & Router        & Classifies query as SQL, RAG, or hybrid; picks execution path \\
  4 & Text2SQL      & Generates SQL against DuckDB schema using Gemini 2.0 Flash \\
  5 & Execute       & Runs query; captures result set or error \\
  6 & Validate      & Checks result for empty set, type errors; loops back to SQL on error \\

\subsection{Competitive Moat}

\begin{enumerate}[leftmargin=1.8em, itemsep=3pt]
  \item \textbf{India-specific regulatory data} --- NSE SAST/PIT and NSDL FII/DII feeds
        not available through generic global APIs.
  \item \textbf{Backtested signals} --- every signal carries a win rate grounded in
        NSE/BSE historical data; LLM-only tools cannot replicate this.

        dashboards that recompute from scratch on every load.

        actual holdings.
  \item \textbf{Compound improvement} --- each interaction improves signal ranking;
        accuracy scales with user base.
\end{enumerate}

\section{Setup \& Quick Start}
% ============================================================

\subsection{Prerequisites}

\begin{itemize}[leftmargin=1.6em, itemsep=2pt]
  \item Python 3.11+ and \texttt{uv} package manager
  \item Node.js 18+ (frontend build)
  \item Google Cloud account with Vertex AI enabled (Gemini 2.0 Flash)
  \item \texttt{NEWS\_API\_KEY} from \texttt{newsapi.org} (required for live news)
  \item 4 GB+ RAM recommended
\end{itemize}

\subsection{Step-by-Step}

\begin{lstlisting}[language=bash]
# 1. Clone
git clone https://github.com/wildcraft958/AlphaStream_India.git
cd AlphaStream_India

# 2. Backend dependencies
cd backend
uv sync

# 3. Configure API keys
cp .env.example .env
# Edit .env: set GOOGLE_APPLICATION_CREDENTIALS, GCP_PROJECT_ID, NEWS_API_KEY

# 4. Initialise DuckDB with Nifty 50 data
uv run python -m src.data.market_schema

# 5. Start backend (cold start ~75 s due to LangChain imports)
./start.sh          # OR: uv run uvicorn src.api.app:app --port 8000

# 6. Frontend (new terminal)
cd ../frontend
npm install && npm run dev

# 7. Open dashboard
# http://localhost:5173
\end{lstlisting}

\subsection{Minimal Mode (No API Keys)}

Set in \texttt{backend/.env}:

\begin{lstlisting}
ENABLE_PATHWAY=false
NEWS_API_KEY=
GCP_PROJECT_ID=
\end{lstlisting}

The UI loads with cached article data and pre-computed recommendations ---
sufficient to evaluate the dashboard, agent outputs, and PDF report generation.
